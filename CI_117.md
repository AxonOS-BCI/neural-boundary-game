# CI 117-lane surface

This repository uses a 117-lane GitHub Actions matrix for fast release-surface validation.

The lanes are intentionally static and deterministic:

- repository identity and required files;
- README / Cargo / web / vectors / release metadata;
- JSON validity;
- shell script hygiene;
- merge-conflict marker scan;
- obvious secret pattern scan;
- oversized checked-in file scan.

Heavy release verification remains available through `scripts/verify_release.sh`.
