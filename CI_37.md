# 37 Foundation Gates

The CI system is a 37-job public release-surface matrix.

The goal is not job-count theatre. Each gate has a narrow responsibility and a written purpose in GitHub Actions summaries.

## Gate groups

| Range | Group | Purpose |
|---:|---|---|
| 01–04 | Identity and Rust foundation | Product identity, version pin, Cargo workspace, lockfile provenance. |
| 05–09 | Legal, security, and release surface | License, commercial terms, security, privacy, release notes. |
| 10–12 | Evidence and machine metadata | Evidence report, source manifest, release.toml. |
| 13–18 | Runtime and data surface | Web entrypoint, PWA manifest, service worker, WASM, replay vectors, JSON validity. |
| 19–22 | Release tooling | Replay validator, version checker, web build, release gate. |
| 23–28 | Hygiene and safety | Stale version scan, token scan, conflict markers, bloat, DOGE safety, script discipline. |
| 29–32 | Documentation and presentation | Policy docs, workflows, preview asset. |
| 33–37 | Foundation Standard | Cognitive sovereignty positioning, Rust/WASM signal, canonical URL, governance brief, CI annotation. |

The heavy release gate remains `scripts/verify_release.sh`.
